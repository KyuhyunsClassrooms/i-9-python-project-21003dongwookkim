# 최종 보고서

## 1. 프로그램 주제와 목적

내 프로그램의 주제:
공식 가이드라인(SRS) 및 Tetr.io 스타일의 물리 법칙을 구현한 텍스트 기반 올스핀 테트리스 게임
이 프로그램을 만든 이유:기존의 단순한 테트리스 프로그램들은 블록이 벽이나 바닥에 조금만 닿아도 회전이 완전히 막히거나 즉시 고정되어 버리는 한계가 있었습니다. 현대 테트리스 유저들이 즐겨 쓰는 고난도 기술인 'T-스핀 트리플(TST)'이나 좁은 틈새를 비비고 들어가는 현란한 스핀 메커니즘을 터미널 환경에 완벽하게 재현하여, 고난도 테트리스의 정교한 물리 법칙과 묵직한 조작 손맛을 구현하고자 이 프로그램을 만들었습니다.


---

## 2. 문제 분해 및 모델링

### 입력
사용자 조작 데이터: 사용자가 키보드로 실시간 입력하는 단일 문자 키값 (A/D: 좌우 이동, S: 소프트 드롭, Space: 하드 드롭, W/E: 시계/반시계 회전, C: 홀드, Q: 게임 종료)

시스템 제어 데이터: 블록이 한 묶음 안에서 치우침 없이 무작위로 등장하도록 제어하는 공식 7-Bag 무작위 배열 데이터 및 자연 낙하 주기를 계산하기 위한 타임스탬프 시간값

### 처리
회전 행렬 및 물리 연산: 3x3 및 4x4 정방형 고정 그리드 행렬의 전치와 반전 연산을 연계한 축 고정 회전 처리 기법을 적용함

SRS 월킥 보정 탐색: 제자리 회전 반경 내에 벽이나 장애물이 감지되면 공식 가이드라인 5단계 오프셋 테이블을 순차 검사하여 미끄러지듯 이동 가능한 좌표로 자동 보정함

판정 및 타이머 제어: T-블록 대각선 모퉁이를 검사하는 3-Corner 룰 알고리즘 연산 및 바닥 접촉 시 유저가 조작하면 고정 타이머를 무한 초기화하는 인피니티 리셋 로직 처리

### 출력
실시간 터미널 화면 렌더링: 가로 10칸, 세로 20칸의 2차원 리스트 맵 상태를 파악하여 빈 공간은 '.', 고정 블록은 '[X]', 제어 중인 블록은 '[O]' 문자로 실시간 출력함

대시보드 및 특수 이벤트 피드백: 실시간 점수, 누적 지운 줄 수, 홀드 슬롯, 다음 블록 5개 대기열 정보와 게임 시작 전 3-2-1 카운트다운 연출 및 TST 달성 알림 문구 출력

## 3. 사용한 2차원 리스트 설명

2차원 리스트 이름:board (전체 게임판 공간 매트릭스 데이터)

한 행의 의미:테트리스 게임판의 가로 1줄(총 너비 10칸의 공간) 격자에 블록이 쌓여있는 실시간 점유 상태를 의미합니다.

각 열의 의미:

| 열 번호 | 의미 | 실제 예시 |
|---:|---|---|
| 0번 열 |게임판 가로 격자에서 맨 왼쪽 첫 번째 칸 (X = 0)| 1 (이미 블록이 쌓여 고정된 상태) |
| 1번 열 |게임판 가로 격자에서 왼쪽에서 두 번째 칸 (X = 1)| 2 (현재 플레이어가 조작 중인 블록이 위치함) |
                     .
                     .
                     .
| 9번 열 | 게임판 가로 격자에서 맨 오른쪽 마지막 열 칸 (X = 9) | 0 (아무것도 없는 빈 공간 상태) |


이 2차원 리스트를 사용한 이유:
가로 격자와 세로 격자의 바둑판 형태로 구성된 테트리스 게임판 구조를 행 인덱스(Y)와 열 인덱스(X)를 가진 2차원 평면 좌표계로 가장 직관적으로 매핑할 수 있기 때문입니다. 또한 가로줄이 모두 가득 찼을 때 해당 행 전체를 삭제하고 새로운 행을 삽입하는 라인 소거 처리를 편리하게 수행할 수 있어 사용했습니다.

## 4. 함수 설명

| 함수 이름 | 역할 | 입력값 | 반환값 또는 출력 |
|rotate_matrix|3x3 또는 4x4 고정 격자 레이아웃 내부에서 중심축 이탈 없이 90도 회전 행렬 연산을 수행함|matrix (2차원 리스트), clockwise (부울값)|중심축 처리가 완료된 새로운 회전 형태의 2차원 리스트 배열|
| rotate_piece | 제자리 회전 및 실패 시 5단계 SRS 월킥 데이터를 순차 연산하여 구멍 사이를 타고 들어가도록 보정함 | clockwise (부울값) | 없음 (내부 인스턴스 변수인 블록 배열, 좌표, 회전 상태 변수를 직접 갱신) |
| check_t_spin_status | T-블록 주변 모퉁이의 누적 차단 값을 추적해 T-스핀 및 T-스핀 트리플(TST) 조건 성립을 실시간 감시함 | kick_used (부울값), kick_index (정수형) | 없음 (판정 만족 시 내부 T-스핀 상태 플래그 활성화 및 특수 전용 알림 문구 업데이트) |
| update_lock_delay_logic | 블록이 바닥에 충돌했을 때 바로 고정되지 않게 유예 타이머를 가동하고, 조작 성공 시 타이머를 리셋함 | 없음 | 없음 (유예 시간 만료 판단 시 블록 잠금 함수인 lock_piece_and_next를 트리거) |
| clear_lines | 완성된 가로 행 데이터를 분석해 소거하고 일반 라인 클리어와 특수 TST 스핀 클리어 점수를 구분해 가산함 | 없음 | 없음 (유저의 현재 누적 점수 및 최종 누적 라인 소거 카운트 데이터 갱신) |

가장 중요한 함수 하나를 골라 자세히 설명하세요.

함수 이름:check_t_spin_status (공식 가이드라인 3-Corner T-Spin 판정 함수)

설명:T-블록이 회전 명령을 성공적으로 완수한 직후 작동하는 물리 판정 함수입니다. T-블록의 중심점을 기준으로 대각선 방향 4곳의 꼭짓점 모퉁이 좌표를 실시간으로 스캔합니다. 이 4곳 중 벽이나 이미 고정되어 있는 장애물 블록으로 가로막힌 곳이 3곳 이상인지를 판정(3-Corner Rule)합니다. 특히 입구가 한 칸 가로막힌 고난도 TST 지형 내부로 5번째 특수 월킥 경로를 타고 빨려 들어온 조건까지 완벽히 추적하여, 라인 클리어 함수와 동기화해 대량의 T-스핀 트리플 점수(1,600점)를 확정 짓는 핵심적인 알고리즘입니다.

## 5. 조건문과 반복문 활용

### 조건문 사용 부분

어떤 조건을 검사했는가?

블록이 좌우, 아래로 이동하거나 회전 오프셋 연산을 수행할 때 보드판 외곽 한계선을 이탈하거나 기존 블록 누적 영역을 침범하지 않는 안전한 상태인지를 충돌 체크 조건식(if not self.check_collision)으로 검사했습니다.

T-블록의 사방 모퉁이 충돌 격자 개수가 3개 이상인지 조건 검사를 거친 후, 라인이 동시에 3줄 소거되었는지를 연쇄 검사하여 TST 점수 가산 조건을 세부 필터링했습니다.

### 반복문 사용 부분

무엇을 반복 처리했는가?

게임오버 플래그 조건이 거짓인 상황 동안 사용자의 비동기 키보드 입력을 상시 대기하고 자연 낙하 시간을 계산하여 화면을 새로 고치는 전체 게임 메인 제어 루프(while)를 반복했습니다.

현재 떨어지고 있는 정방형 블록 조작 매트릭스(SHAPES) 내부의 유효한 블록 격자(1) 데이터들의 상대적 인덱스 위치를 실시간으로 파악하여 전체 맵 배열에 합성하기 위해 2중 반복문(for r, for c)을 순회했습니다.

### 조건문과 반복문이 함께 작동한 부분

가로줄 제거 기능(clear_lines) 로직에서 함께 작동했습니다. 20칸 높이의 게임판 전체 구조를 반복문(for)으로 위에서 아래로 한 줄씩 차례대로 검사하면서, 해당 행 내부의 가로 10칸 격자 데이터가 단 하나의 빈칸도 없이 모두 블록(1)으로만 가득 찼는지에 대한 일치성 조건을 다중 조건식(if not all)으로 검사하여 참일 때만 해당 줄을 폭파하고 상단에 빈 행을 추가하는 연산을 수행했습니다.

## 7. 테스트와 오류 수정

### 테스트 1: 정상 사례

입력:블록이 공중에 정상 스폰되어 낙하하는 도중 키보드 좌측 이동 키(A) 또는 우측 이동 키(D) 입력

예상 결과:블록이 벽을 뚫지 않는 선에서 사용자가 누른 입력 방향 그대로 가로 격자 인덱스를 한 칸 이동하고 실시간 렌더링 화면에 갱신되어 출력되어야 함

실제 결과:양쪽 좌우 벽면에 충돌하기 전까지 걸림돌 없이 부드럽게 실시간 좌우 수평 이동이 올바르게 실행됨


### 테스트 2: 예외 또는 경계 사례

입력:블록이 바닥이나 적재된 블록 더미 표면에 정면으로 완전히 맞닿아 정지한 경계 상태에서 회전 키(W/E) 또는 좌우 이동 키를 빠르게 난사 조작함

예상 결과:바닥에 닿았을 때 즉시 잠금 고정되지 않고 테트리오 사양에 준하는 0.5초의 낙하 유예 타이머가 기동되어야 하며, 유저가 좌우로 움직이거나 회전시키는 액션이 성공할 때마다 유예 시간이 처음부터 다시 리셋(Infinity Reset)되어 비비기 조작 시간을 보장해야 함

실제 결과:바닥에 블록이 닿은 경계면 상태에서도 바로 고정되지 않고 연타 조작을 수행하는 동안 계속 굳지 않은 채 부드럽게 이동 및 회전하여 빈 구멍 속 틈새로 안정적으로 삽입 조작이 완료되었으며, 조작을 멈추고 가만히 대기했을 때는 정확히 0.5초 뒤 완벽하게 잠금 고정 처리가 됨


### 발견한 오류와 수정한 방법

초기 프로토타입 설계 시 T-블록이 회전할 때마다 가로 레이어 행 배열의 내부 크기가 유동적으로 반전 및 변환되어 매트릭스의 고정 중심점이 이탈하고, 이로 인해 SRS 월킥 테이블 데이터의 기준 오프셋 좌표 축이 완전히 엇나가 벽을 뚫거나 이상한 공중으로 순간 이동하는 심각한 궤도 이탈 오류를 발견했습니다.

이를 수정하기 위해 블록 구조를 동적 배열 크기가 아닌 T-미노 기준 무조건 3x3 격자의 고정 크기 레이아웃 행렬 구조로 데이터 매트릭스를 전면 리팩토링했습니다. 이후 행렬의 행과 열을 뒤바꾸는 전치 공식과 행 순서를 거꾸로 뒤집는 반전 로직을 결합한 수학적 행렬 회전 함수(rotate_matrix)를 별도로 완전히 격리 분리해 냄으로써, 어떤 회전 상태에서도 기준 중심축이 영구히 고정되도록 완벽히 수정했습니다.

## 8. Gemini 활용 내용 요약

Gemini에게 도움받은 부분:
공식 가이드라인 표준 규격의 SRS 월킥 5단계 탐색 좌표 매핑 테이블을 구현하기 위한 파이썬 딕셔너리 구조 오프셋 설계 도움

바닥 접촉을 감시하여 조작 발생 시 타이머를 처음 상태로 완전히 갱신시켜 주는 인피니티 리셋 유예 시간(Lock Delay) 메커니즘 코드 모델링 도움

단순 주변 사방 탐색을 넘어 대각선 꼭짓점 조건식과 특수 킥 트래킹 조건을 결합한 정교한 TST(T-스핀 트리플) 3-Corner 룰 판정 핵심 알고리즘 작성 지원


Gemini가 제안했지만 내가 수정하거나 사용하지 않은 부분:
Gemini는 시각적 화려함을 극대화하기 위해 초기에 파이썬 외부 GUI 그래픽 윈도우 라이브러리인 'Tkinter'를 기반으로 별도의 창을 띄워 게임 화면을 그리는 구조를 제안했었습니다. 하지만 저는 학교 제출용 보고서의 핵심 요구 조건이자 2차원 리스트의 순수한 공간 모델링 데이터 조작을 가장 정석대로 명확히 보여주기 위해, GUI 그래픽 방식 제안을 과감히 거절하고 sys.stdout을 활용해 2차원 문자열 보드를 리프레시하여 터미널 콘솔창에 띄우는 완전한 텍스트 기반 렌더링 시스템으로 구조를 직접 수정하여 개발했습니다.


최종 코드에서 내가 직접 판단하거나 수정한 부분:
터미널 텍스트 환경 특유의 키보드 입력 버퍼 지연 현상으로 인해 블록 조작이 밀리거나 화면이 마구 깜빡거리는 예외적 병목 연산을 해결하기 위해, 메인 루프 내부의 대기 타임 주기를 마이크로 초 단위인 time.sleep(0.01) 상수로 직접 실험해가며 가장 부드러운 반응 속도로 정밀 튜닝했습니다.

또한 게임 시작 전 카운트다운의 연출 시각적 시간 간격을 학교 발표 스피치 템포와 긴장감 유도 주기에 딱 알맞도록 기본 제안값인 1초에서 0.7초 단위 간격으로 손수 세부 수동 조정을 거쳐 적용했습니다.

## 9. 한계와 개선 방안

현재 프로그램의 한계:
텍스트 문자를 기반으로 가동되는 콘솔 터미널 실행 환경의 플랫폼 한계성 때문에, 현대적인 테트리스 게임들이 보여주는 화려한 파티클 이펙트나 블록이 지워질 때의 부드러운 애니메이션 효과를 시각적으로 연출하기 어렵습니다. 또한 블록의 종류별 고유 컬러(Color)가 구현되어 있지 않아 모든 적재 블록이 동일한 형태의 텍스트 기호로 표현되어 직관성이 다소 떨어집니다.

다시 개선한다면 추가하고 싶은 기능:
ANSI Escape 코드를 출력 문자열 내부에 결합하는 색상 시스템 코드를 설계하여 I-블록은 하늘색, T-블록은 보라색 등 공식 가이드라인 표준 RGB 컬러 색상을 터미널에 완벽히 구현하고 싶습니다. 또한 플레이어가 지운 총 누적 줄 수의 임계치에 도달할 때마다 시스템의 기본 자연 낙하 속도가 단계적으로 빨라지는 '레벨 업 난이도 밸런싱 시스템'과 연속으로 라인을 연속 돌파할 때 터지는 '콤보(Combo) 점수 배율 시스템'을 탑재하여 완벽한 상용 게임 수준으로 발전시키고 싶습니다.
